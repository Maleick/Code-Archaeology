Why it exists

Technical debt cleanup with an audit trail.

Start with evidence. Review the site report. Restore only what can be verified.

Catalog

Find the buried layers

Dead code, legacy fallbacks, circular dependencies, weak types, duplication, and defensive slop are cataloged before changes happen.

Review

Separate reports from edits

survey mode writes .archaeology/ reports only. excavate adds mock patches for review. restore is the only mode that edits source files.

Verify

Gate every restoration

Restore phases run tests and type checks between expeditions, then revert and stop if a phase introduces failures.

Expedition map

From survey to final catalog.

The order is fixed so cleanup does not hide evidence or break later analysis.

Safety model

Reports first. Edits last.

Code Archaeology treats your repository like a site under excavation. It records evidence, asks for review, and only modifies source files when restore mode is deliberately selected.

01

Survey

Generate .archaeology/ reports without touching source files.

02

Excavate

Create reports and mock patches so humans can inspect intended changes.

03

Restore

Apply approved changes on an isolated branch with test and type-check gates.

04

Revert on failure

If verification fails, the phase is reverted and the expedition stops.
